data_models module¶

This file contains the models used for structuring data in gene expression data

class data_models.clinical_data(id, other_fields)[source]¶

Represents clinical data about each person where a gene sample was collected from

Parameters:
  • id (int) – represents the unique identifier given to the person this sample was taken from
  • other_fields (dict) – the fields associated with this person. This takes the form of a map where the name of each additional field is a key for the value of the field. The map should have strings as keys going to objects as values
class data_models.expression_profile(id, gene, intensity, other_fields)[source]¶

Represents a single expression value of a single gene from a single sample.

Parameters:
  • id (int) – represents the unique identifier given to the person this sample was taken from
  • gene (str) – the name of the gene whose expression is recorded in this profile
  • intensity (float) – the expression levels of the gene
  • other_fields (dict) – other fields associated with this profile. This takes the form of a map where the name of each additional field is a key for the value of the field. The map should have strings as keys going to objects as values
class data_models.gene_set(set_name, url, genes)[source]¶

Represents an a priori gene set taken from the Broad institute’s molecular signatures database

Parameters:
  • set_name (str) – the name of this set
  • url (str) – the MSigDB webpage associated with this gene set
  • genes (set) – a set of genes which are located in this set
class data_models.sample(profiles, id)[source]¶

A grouping of gene expression profiles under the person who gave the gene sample

Parameters:
  • profiles (dict) – A mapping of gene names to expression_profiles
  • id (int) – the unique identifier for the person who gave samples which were used to derive the expression_profiles
